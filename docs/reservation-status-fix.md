# 予約カレンダー監視メモ

## 概要
Google Cloud Run 上で Puppeteer を利用して予約ページのカレンダーを監視する際、iframe 内の `td` 要素が検出できない問題が発生していました。`waitForCalendarContext` の導入と、メインページと iframe の両方を順番に探索するロジックの追加により、問題が解消されたことを確認しました。

## 対応内容
- 予約ページの URL を `.../fujimino` まで含めるように修正し、直接カレンダーが表示されるページに遷移するよう調整。
- カレンダーを探索する際、メインページと iframe の両方で `td` 要素を検出できるように待機処理を改善。
- ログ出力を拡充し、カレンダーがどのコンテキストで検出されたかを明確化。

## 今後の運用
- 監視が安定していることを定期的に確認する。
- DOM 構造に変更があった場合は `waitForCalendarContext` のセレクタリストを更新する。

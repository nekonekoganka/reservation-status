# Chrome Web Store 公開手順書

このガイドでは、「予約状況チェッカー」をChrome Web Storeに**非公開**で公開する手順を説明します。

---

## 📋 事前準備

### 必要なもの

- ✅ Googleアカウント
- ✅ クレジットカード（5ドルの登録料支払い用、初回のみ）
- ✅ `reservation-checker-v1.1.zip`（すでに作成済み）
- ✅ スクリーンショット画像（1280x800、最低1枚）
- ✅ プライバシーポリシー（`privacy-policy.md` を参照）

---

## ステップ1: Chrome Web Store Developer Dashboard にアクセス

1. **Chrome Web Store Developer Dashboard** を開く
   - URL: https://chrome.google.com/webstore/devconsole

2. Googleアカウントでログイン

3. **初回のみ:** 開発者登録
   - 「開発者として登録」をクリック
   - 利用規約に同意
   - **5ドルの登録料を支払い**（クレジットカード必須）
   - 支払い完了後、Developer Dashboard にアクセス可能

---

## ステップ2: 新しいアイテムを作成

1. Developer Dashboard で **「新しいアイテム」** ボタンをクリック

2. **zipファイルをアップロード**
   - `reservation-checker-v1.1.zip` を選択
   - 「アップロード」をクリック

3. アップロード完了を待つ（数秒〜1分）

---

## ステップ3: ストア掲載情報を入力

アップロード後、以下の情報を入力します。

### 3-1. 基本情報

**拡張機能名:**
```
予約状況チェッカー
```

**概要（短い説明）:**
```
予約ページの空き状況を確認するツール
```

**詳細な説明:**
```
予約状況チェッカーは、オンライン予約ページの空き状況を簡単に確認できるChrome拡張機能です。

主な機能:
- ボタンをクリックするだけで予約状況を自動チェック
- ページを自動更新して最新の情報を取得
- 予約可能な日時を素早く確認

使い方:
1. 対象の予約ページを開く
2. 画面右下に表示される「🔄 予約状況を更新する」ボタンをクリック
3. ページが自動更新され、数秒後に予約状況が確認されます

特徴:
- シンプルで使いやすいインターフェース
- 自動更新機能で最新情報を取得
- 個人利用に最適化
```

### 3-2. カテゴリ

**カテゴリ:**
- 「生産性向上」を選択

### 3-3. 言語

**言語:**
- 「日本語」を選択

---

## ステップ4: スクリーンショットをアップロード

1. **「スクリーンショット」** セクションに移動

2. **スクリーンショットをアップロード**
   - 「画像を追加」をクリック
   - 作成したスクリーンショット画像（1280x800）を選択
   - 最低1枚、最大5枚まで追加可能

3. （オプション）キャプションを追加
   - 例: 「予約状況を自動チェック」

---

## ステップ5: プライバシー設定

### 5-1. プライバシーポリシー

1. **「プライバシーのプラクティス」** セクションに移動

2. **プライバシーポリシーのURL** を入力
   - オプション1: GitHubリポジトリのプライバシーポリシーを公開してURLを入力
   - オプション2: 簡易的なテキストで説明（以下のテンプレート使用）

**プライバシーポリシーのテキスト例:**
```
本拡張機能は、個人情報を一切収集しません。
予約ページのDOM情報を読み取り、予約状況（空きあり/満）のみを処理します。
収集したデータは第三者と共有されません。
```

### 5-2. データの使用

**「ユーザーデータを収集しますか？」**
- 「いいえ」を選択

---

## ステップ6: 公開範囲を設定

**重要:** ここで**非公開**に設定します。

1. **「配布」** または **「公開設定」** セクションに移動

2. **公開範囲** を選択
   - ✅ **「非公開」** を選択
   - または **「信頼できるテスター」** を選択

3. **公開対象** を設定
   - 「自分のみ」または「特定のメールアドレス」を入力
   - 自分のGoogleアカウントのメールアドレスを入力

4. 設定を保存

---

## ステップ7: 審査申請

1. すべての必須項目が入力されているか確認

2. **「審査のため送信」** または **「公開」** ボタンをクリック

3. 確認ダイアログで **「送信」** をクリック

---

## ステップ8: 審査を待つ

**審査期間:**
- 通常: 1〜3営業日
- 長い場合: 1週間程度

**審査中の表示:**
- Status: 「審査中」または「Pending Review」

**審査完了の通知:**
- 登録したメールアドレスに通知が届きます

---

## ステップ9: 拡張機能をインストール

審査が完了したら、以下の手順でインストールします。

### 9-1. 現在の拡張機能を削除

1. `chrome://extensions/` を開く

2. 現在の「予約状況自動更新」または「予約状況チェッカー」を探す

3. **「削除」** をクリック

### 9-2. Chrome Web Storeからインストール

1. **Developer Dashboard** で公開した拡張機能のページを開く

2. **「ストアで表示」** または **「View in Store」** リンクをクリック

3. Chrome Web Storeのページが開く（非公開なので自分しか見えません）

4. **「Chromeに追加」** ボタンをクリック

5. 確認ダイアログで **「拡張機能を追加」** をクリック

6. インストール完了！

---

## ✅ 完了確認

### インストールが成功したか確認

1. `chrome://extensions/` を開く

2. 「予約状況チェッカー」が表示されているか確認

3. **デベロッパーモード** がOFFでも動作することを確認

4. 予約ページを開いて、ボタンが表示されるか確認

---

## 🔧 トラブルシューティング

### 審査が却下された場合

**よくある却下理由:**
1. プライバシーポリシーが不十分
2. スクリーンショットが不足
3. 説明文が不明瞭

**対処方法:**
- 却下理由をメールで確認
- 指摘された箇所を修正
- 再度審査申請

### インストールできない場合

**原因1: 非公開設定が正しくない**
- Developer Dashboard で公開範囲を再確認
- 自分のメールアドレスが含まれているか確認

**原因2: ログインアカウントが異なる**
- Chrome Web Storeで使用しているGoogleアカウントを確認
- Developer Dashboard で登録したアカウントと同じか確認

## 📝 今後の更新方法

拡張機能を更新する場合:

1. `manifest.json` の `version` を上げる（例: 1.1 → 1.2）

2. 新しいzipファイルを作成

3. Developer Dashboard で **「パッケージを更新」** をクリック

4. 新しいzipファイルをアップロード

5. 変更内容を説明

6. 審査申請
